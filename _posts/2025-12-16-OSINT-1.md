## 1. I'm Searching for Person
Google Search Engine에서 사람을 한번 찾아보자.

> **\<Target info\> (가명)** <br>
> Name: Gomtang <br>
Works at: Udemy

해당 인물에 대해 가능한 많은 정보를 수집하는 것이 우리의 목표다.

Google에 이름을 그대로 검색해보면,

![img](https://github.com/sari-harin/blog-imgs/blob/main/OSINT2-1.png?raw=true)

검색 결과가 많다. 하나하나 찾아볼 수는 없으므로 우리는 Google Dorks를 사용한다.

**Google Dorks란?:** 일반적인 Search로는 접근이 어려운 민감한 정보에 접근할 수 있게 해주는 Search Operators

### a. " " / Double Quotation Mark
`"Gomtang"`

정확히 일치하는 결과만을 돌려준다.

![img](https://github.com/sari-harin/blog-imgs/blob/main/OSINT2-2.png?raw=true)
_검색 결과가 확연히 줄었다._

우리는 해당 인물의 근무지까지 알고 있으므로, 다음과 같이 narrow down한 검색을 할 수 있다.

`"Gomtang" "Udemy"`

### b. Verbatim Search
`Search tools → All results → Verbatim`

`도구 → 모든 결과 → 완전 일치`

큰따옴표처럼 정확한 검색을 지원해준다.

### c. URL Clean
[https://urlclean.com/](https://urlclean.com/)

Google에서 검색한 URL을 깔끔하게 정리해준다. Google 검색 결과에서 복사한 URL에는 **불필요한 추적 파라미터**가 포함되어 있기 때문.

이렇게 정리된 URL을 기록해두면 추후 추적이 쉽다.

### d. Search for face
`Images → Search tools → Type → Clip art`

`이미지 → 도구 → 유형 → 클립아트`

그리고 상단 URL에서 `tbs=itp:clipart`를 찾아 `clipart`를 `face`로 바꿔준다.

### e. Cached page
`cache:[Cleaned-URL]`

페이지가 더이상 접근 불가능할 경우 구글에 Cache된 페이지를 확인한다.

e.g. 판매 완료된 상품 페이지의 캐시된 버전 확인

### f. inurl
`inurl:[keyword]`

url에 키워드가 존재하는 결과만 반환한다. Username을 검색하는 데에 사용할 수 있다.

### g. OR
`site:[site] OR site:[site] "[searching-keyword]`

OR은 대문자로 작성되어야 한다.

### h. intitle
`intitle:[keyword]`

페이지의 제목에 키워드가 들어간 경우만 반환한다.

> keyword에 띄어쓰기가 포함되어 있을 경우, 띄어쓰기는 .으로 대체한다.
e.g. gom.tang

### i. * / Asterisk
`[keyword-1] * [keyword-2]`

일종의 와일드카드. 인물의 미들네임을 찾는 데에 활용할 수 있다.

### j. - / Hyphen
`"[keyword]" -site:[site]`
`"[keyword]" -"[exclude-keyword]"`

특정 검색어를 제외한 결과만을 보여준다.

### k. Site
`"[keyword]" site:[site]`

사이트를 지정하여, 원하는 사이트에서의 키워드 검색 결과만 볼 수 있다.

### l. Filetype
`filetype:[]`

지정한 파일 확장자에 해당하는 검색 결과만 보여준다. 가능한 파일 타입은 [이 링크](https://developers.google.com/search/docs/crawling-indexing/indexable-file-types)에서 확인해볼 수 있다.
사이트를 회사 사이트로, 파일 타입을 pdf로 지정하면 회사의 문서를 쉽게 찾아볼 수 있겠다.

### l-1. File metadata
위에서 찾은 PDF 파일을 PDF Metadata를 확인할 수 있는 [사이트](https://www.pdfyeah.com/view-pdf-metadata/)에 넣어 확인해보면 
PDF 파일을 오래된 SW로 만들었다면 취약점 exploit도 가능해진다.

## 2. 국가별 검색 결과 제한 우회
국가의 규정에 따라 볼 수 없는 검색 결과가 있을 수 있다. 우회하는 방법은,
1. VPN 사용
  - 이 경우 google은 VPN 사용 여부를 알고 있으므로 capcha를 요구한다.
2. SquareX: Browser extension
  - Disposable Browser라는 기능을 제공한다. [이 링크](https://chromewebstore.google.com/detail/squarex-be-secure-anonymo/kapjaoifikajdcdehfdlmojlepfpkpoe)에서 사용하면 되나, 한국에서는 접속이 안 되는 것 같다.

## 3. Google Hacking Database / GHDB
[GHDB](https://www.exploit-db.com/google-hacking-database)는 우리가 앞서 확인한 search operators를 지원하며, 취약한 서버나 패스워드, 유저네임 등 민감한 정보를 검색 가능하게 한다.

### Example: Searching Public Cameras
1. Quick Search->Camera 서치
2. 적절한 Dork(`inurl:"view.shtml" "camera"`)를 복사.
3. Google 검색창에 사용

발견한 이미지를 메타데이터 확인 [사이트](https://exif.tools/)에 넣어 정보 확인도 가능하다. 사진이 어디서 찍혔는지, 위치 정보까지 확인 가능하다.

## 4. DorkGPT
[DokrGPT](https://www.dorkgpt.com/)는 Google Dorks를 AI로 생성해준다.

e.g. Search for a Gomtang in a pdf file을 검색하면, 적절한 Google Dorks를 생성해준다.



## 3. Searching . . .
가장 먼저 `" "`를 활용해 찾으려는 인물의 추가 정보를 알아낸다.

예를 들자면, 링크드인에 들어가 프로필 이미지, 또 다른 계정 ID, 직책, 국가와 같은 추가 정보를 얻어낼 수 있다.

얻은 정보들에 관해 Note를 작성하는데, SNS나 웹사이트 프로필의 링크를 기록해둔다. 나는 노션을 사용한다.

이후, 이름과 다른 정보(직책, 국가 등)를 `" "`으로 묶어 검색해본다.

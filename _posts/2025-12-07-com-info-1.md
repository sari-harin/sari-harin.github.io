## 📌 개요

- **정의**:
    
    공격에 대한 간략한 정의를 작성합니다.
    
- **목적**:
    
    공격자가 이 공격을 통해 노리는 목적은 무엇인지 서술합니다.
    

## 🧠 작동 원리

- 공격이 어떻게 동작하는지, 내부 동작 방식이나 로직을 설명합니다. (기술적 배경, 흐름 또는 매커니즘)
- 예: SQL 인젝션은 사용자의 입력이 검증되지 않은 채로 SQL 쿼리에 포함되어 실행되는 취약점입니다.

## 🔥 공격 시나리오

- 현실적인 예시나 시나리오 작성 (가능하면 코드 포함)(예시 코드)

```sql
-- SQL Injection 예시
SELECT * FROM users WHERE username = 'admin' AND password = '1234' OR '1'='1';
```

- 코드나 상황에 대한 해설 포함, 어떤 식으로 공격이 실행되는지 구체적인 설명
